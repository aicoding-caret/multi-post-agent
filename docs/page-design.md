# 페이지 디자인 요구사항 (교육용 웹 애플리케이션)

## 1. 개요

본 문서는 "AI 콘텐츠 변환 및 Blogger 포스팅 에이전트" 교육용 웹 애플리케이션의 사용자 인터페이스(UI) 디자인에 대한 기본 요구사항을 정의합니다. 사용자가 쉽게 콘텐츠를 작성하고, 변환하며, Blogger에 포스팅할 수 있도록 직관적이고 간결한 디자인을 목표로 합니다.

## 2. 전체 레이아웃

* **단일 페이지 애플리케이션 (SPA) 형태**로 구성하여 사용자의 작업 흐름이 끊기지 않도록 합니다.
* **반응형 디자인**을 고려하여 다양한 화면 크기에서도 기본적인 사용성을 제공합니다. (교육 범위 내에서는 데스크탑 우선으로 단순화)
* 상단에는 애플리케이션 제목 또는 로고를 표시합니다.

## 3. 주요 페이지 구성 요소

### 3.1. 입력 영역

* **제목 입력 필드:**
    * 포스팅할 콘텐츠의 제목을 입력받는 텍스트 필드.
    * 명확한 레이블("제목")을 표시합니다.
* **내용 입력 영역:**
    * 포스팅할 콘텐츠의 본문을 입력받는 여러 줄 텍스트 영역(Textarea).
    * 명확한 레이블("내용")을 표시합니다.
    * (선택 사항) 간단한 마크다운 입력 지원 또는 위지윅 에디터 (교육 범위에서는 단순 Textarea 우선).

### 3.2. 처리 옵션 영역

* **처리 옵션 선택:**
    * "자동 개시"와 "검토 후 개시" 중 하나를 선택할 수 있는 라디오 버튼 또는 드롭다운 메뉴.
    * 명확한 레이블("처리 옵션")을 표시합니다.
    * 기본값은 "검토 후 개시"로 설정합니다.

### 3.3. 실행 버튼 영역

* **"변환 및 처리 요청" 버튼:**
    * 사용자가 입력한 초안과 선택한 처리 옵션을 서버로 전송하여 AI 에이전트 처리를 시작하는 버튼.
    * 버튼 텍스트는 명확하게 "변환 및 처리 요청" 또는 "시작하기" 등으로 합니다.
    * 처리 중에는 버튼이 비활성화되거나 로딩 상태를 표시합니다.

### 3.4. 결과 표시 및 검토 영역

* **상태 메시지 표시 영역:**
    * AI 에이전트의 처리 상태(예: "변환 중...", "Blogger에 포스팅 중...", "오류 발생: ...") 및 최종 결과(예: "Blogger에 성공적으로 게시되었습니다! URL: [링크]")를 사용자에게 명확하게 알려주는 텍스트 영역.
    * 메시지 유형(정보, 성공, 경고, 오류)에 따라 적절한 스타일(예: Bootstrap Alert 컴포넌트 활용)을 적용합니다.
* **포스팅 URL 표시 영역 (성공 시):**
    * Blogger 포스팅 성공 시, 게시된 글의 URL을 클릭 가능한 링크 형태로 명확하게 표시하는 영역. (예: "포스팅 URL: <a href='...' target='_blank'>...</a>")
    * 이 영역은 포스팅 성공 시에만 나타납니다.
* **변환된 콘텐츠 표시 영역:**
    * AI 에이전트가 변환한 콘텐츠(또는 수정 제안된 콘텐츠)를 보여주는 읽기 전용 텍스트 영역.
    * "검토 후 개시" 옵션 선택 시 이 영역이 활성화됩니다.
* **(검토 시) 수정 의견 입력 영역:**
    * "검토 후 개시" 옵션 선택 시, 사용자가 변환된 콘텐츠에 대한 간단한 수정 의견을 입력할 수 있는 텍스트 영역.
    * 명확한 레이블("수정 의견 (선택 사항)")을 표시합니다.
* **(검토 시) "의견 반영하여 재처리" 버튼:**
    * 사용자가 입력한 수정 의견을 반영하여 AI 에이전트가 콘텐츠를 다시 처리하도록 요청하는 버튼.
* **(검토 시) "최종 배포" 버튼:**
    * 사용자가 변환된 콘텐츠를 최종 확인하고 Blogger에 포스팅하도록 승인하는 버튼.
    * 이 버튼은 "검토 후 개시" 옵션을 선택하고, AI가 변환한 콘텐츠가 표시되었을 때 활성화됩니다.

## 4. 스타일 가이드 (간단 제안)

* **색상:** 주조색, 보조색, 강조색을 정하여 일관성 있게 사용합니다. (예: 파란색 계열의 주조색, 회색 계열의 보조색)
* **폰트:** 가독성이 좋은 기본 웹 폰트를 사용합니다. (예: Open Sans, Noto Sans KR)
* **아이콘:** 필요한 경우 명확한 의미를 전달하는 아이콘을 사용합니다. (교육 범위에서는 최소화)
* **전반적인 느낌:** 깔끔하고 사용하기 쉬운 미니멀한 디자인을 지향합니다.
